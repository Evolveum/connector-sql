== Overview

This guide helps you migrate from custom SQL implementations and other SQL connectors to the SQL Declarative Connector Framework. The framework provides a higher-level, configuration-driven approach to building SQL-based ConnId connectors.

== Understanding the Framework Difference

=== Old Approach: Manual JDBC

In traditional implementations, you manually managed:

* JDBC connections
* Prepared statements
* Result set iteration
* Type conversion
* Exception handling
* Resource cleanup

[source,groovy]
----
// Old: Manual JDBC
Connection conn = dataSource.getConnection();
PreparedStatement stmt = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
stmt.setInt(1, userId);
ResultSet rs = stmt.executeQuery();
List<User> results = new ArrayList<>();
while (rs.next()) {
    User user = new User();
    user.setId(rs.getInt("id"));
    user.setUsername(rs.getString("username"));
    user.setEmail(rs.getString("email"));
    user.setActive(rs.getBoolean("is_active"));
    results.add(user);
}
rs.close();
stmt.close();
conn.close();
----

=== New Approach: Declarative

The framework handles all JDBC operations automatically:

* Connection management (pooling, lifecycle)
* Statement preparation and execution
* Result mapping to ConnId objects
* Pagination and batch operations
* SQL dialect adaptation

[source,groovy]
----
// New: Declarative
objectClass("User") {
    table("users")
    search {
        key("id")
    }
    attribute("id") {
        sql { type INTEGER; primaryKey true }
    }
    attribute("username") {
        sql { type VARCHAR(255) }
    }
    attribute("email") {
        sql { type VARCHAR(255) }
    }
    attribute("is_active") {
        sql { type BOOLEAN }
    }
}
// Framework handles connection, statement, result mapping
----

=== Code Conversion Example

==== Before: Custom JDBC Search

[source,java]
----
public List<ConnectorObject> search(String table, String whereClause, List<Object> params) {
    Connection conn = null;
    PreparedStatement stmt = null;
    ResultSet rs = null;
    try {
        conn = dataSource.getConnection();
        String sql = "SELECT id, username, email, is_active FROM " + table + " WHERE " + whereClause;
        stmt = conn.prepareStatement(sql);
        for (int i = 0; i < params.size(); i++) {
            stmt.setObject(i + 1, params.get(i));
        }
        rs = stmt.executeQuery();
        List<ConnectorObject> results = new ArrayList<>();
        while (rs.next()) {
            ConnectorObject obj = new ConnectorObject();
            obj.setUid(new Uid(rs.getInt("id")));
            obj.addAttribute(new Attribute.Builder("username").setValue(rs.getString("username")).build());
            obj.addAttribute(new Attribute.Builder("email").setValue(rs.getString("email")).build());
            obj.addAttribute(new Attribute.Builder("is_active").setValue(rs.getBoolean("is_active")).build());
            results.add(obj);
        }
        return results;
    } catch (SQLException e) {
        throw new ConnectionFailedException("Search failed", e);
    } finally {
        try { if (rs != null) rs.close(); } catch (SQLException ignored) {}
        try { if (stmt != null) stmt.close(); } catch (SQLException ignored) {}
        try { if (conn != null) conn.close(); } catch (SQLException ignored) {}
    }
}
----

==== After: Declarative Configuration

[source,groovy]
----
objectClass("User") {
    table("users")
    search {
        key("id")
        attribute("username")
        attribute("email")
        attribute("is_active")
    }
}
----

The framework automatically generates search handlers for all filter types.

== Migration Path 3: From Raw SQL-Based Connectors

=== Raw SQL Pattern

Many connectors embed SQL strings directly in Java code:

[source,java]
----
public class UserConnector {
    private DataSource dataSource;
    
    public UserConnector(DataSource ds) {
        this.dataSource = ds;
    }
    
    public User findUser(int id) throws SQLException {
        String sql = "SELECT id, username, email FROM users WHERE id = ?";
        Connection conn = dataSource.getConnection();
        try {
            PreparedStatement stmt = conn.prepareStatement(sql);
            stmt.setInt(1, id);
            ResultSet rs = stmt.executeQuery();
            if (rs.next()) {
                User user = new User();
                user.setId(rs.getInt("id"));
                user.setUsername(rs.getString("username"));
                user.setEmail(rs.getString("email"));
                return user;
            }
            return null;
        } finally {
            conn.close();
        }
    }
    
    public void createUser(User user) throws SQLException {
        String sql = "INSERT INTO users (username, email) VALUES (?, ?)";
        Connection conn = dataSource.getConnection();
        try {
            PreparedStatement stmt = conn.prepareStatement(sql);
            stmt.setString(1, user.getUsername());
            stmt.setString(2, user.getEmail());
            stmt.executeUpdate();
        } finally {
            conn.close();
        }
    }
    
    public void updateUser(User user) throws SQLException {
        String sql = "UPDATE users SET username = ?, email = ? WHERE id = ?";
        Connection conn = dataSource.getConnection();
        try {
            PreparedStatement stmt = conn.prepareStatement(sql);
            stmt.setString(1, user.getUsername());
            stmt.setString(2, user.getEmail());
            stmt.setInt(3, user.getId());
            stmt.executeUpdate();
        } finally {
            conn.close();
        }
    }
    
    public void deleteUser(int id) throws SQLException {
        String sql = "DELETE FROM users WHERE id = ?";
        Connection conn = dataSource.getConnection();
        try {
            PreparedStatement stmt = conn.prepareStatement(sql);

== Migration Path 4: From Hand-Written SQL Connection Code

=== Hand-Written Pattern

[source,java]
----
public class SqlConnector {
    private String jdbcUrl;
    private String username;
    private String password;
    private Connection connection;
    
    public SqlConnector(String url, String user, String pass) {
        this.jdbcUrl = url;
        this.username = user;
        this.password = pass;
    }
    
    public void connect() throws SQLException {
        connection = DriverManager.getConnection(jdbcUrl, username, password);
    }
    
    public void disconnect() throws SQLException {
        if (connection != null && !connection.isClosed()) {
            connection.close();
        }
    }
    
    public List<Map<String, Object>> executeQuery(String sql, List<Object> params) {
        List<Map<String, Object>> results = new ArrayList<>();
        try (PreparedStatement stmt = connection.prepareStatement(sql)) {
            for (int i = 0; i < params.size(); i++) {
                stmt.setObject(i + 1, params.get(i));
            }
            ResultSet rs = stmt.executeQuery();
            ResultSetMetaData meta = rs.getMetaData();
            int colCount = meta.getColumnCount();
            while (rs.next()) {
                Map<String, Object> row = new HashMap<>();
                for (int i = 1; i <= colCount; i++) {
                    row.put(meta.getColumnName(i), rs.getObject(i));
                }
                results.add(row);
            }
        } catch (SQLException e) {
            throw new RuntimeException("Query failed", e);
        }
        return results;
    }
}
----

=== Declarative Framework Approach

[source,groovy]
----

        jdbcUrl = "jdbc:postgresql://localhost:5432/mydb"

        objectClass("User") {
            table("users")
            attribute("id") { sql { type INTEGER; primaryKey true } }
            attribute("username") { sql { type VARCHAR(255) } }
        }
    }
}

== Type Mapping Differences

=== SQL Type to Java/Connector Type Mapping

| SQL Type | Java Type | Connector Type | Notes |
|-----|-----|-----|-----|
| `VARCHAR(n)` | `String` | String | Character data |
| `NVARCHAR(n)` | `String` | String | Unicode character data |
| `INTEGER` | `Integer` | Integer | 32-bit integers |
| `BIGINT` | `Long` | Integer | 64-bit integers |
| `SMALLINT` | `Short` | Integer | 16-bit integers |
| `TINYINT` | `Byte` | Integer | 8-bit integers |
| `BOOLEAN` / ` BIT` | `Boolean` | Boolean | True/false values |
| `DATE` | `java.sql.Date` | Date | Date only |
| `TIME` | `java.sql.Time` | Date | Time only |
| `TIMESTAMP` | `java.sql.Timestamp` | Date | Date and time |
| `DECIMAL(p,s)` | `BigDecimal` | Decimal | Precise decimals |
| `NUMERIC(p,s)` | `BigDecimal` | Decimal | Same as DECIMAL |
| `FLOAT` | `Float` | Double | Single precision |
| `DOUBLE` | `Double` | Double | Double precision |
| `BLOB` | `byte[]` | Binary | Binary large object |
| `CLOB` | `String` | String | Character large object |

=== Migration Considerations

1. **Precision loss**: `BIGINT` to `Integer` may cause overflow
2. **Type coercion**: SQL `VARCHAR` may need conversion to `INTEGER`
3. **Null handling**: SQL NULL vs Java null vs Connector null
4. **Precision**: `FLOAT` vs `DOUBLE` precision differences

=== Custom Type Handlers

[source,groovy]
----
objectClass("User") {
    table("users")

== SQL Dialect Differences

=== Supported Dialects

The framework supports multiple SQL dialects automatically:

| Database | Dialect Name | Key Features |
|-----|-----|-----|
| PostgreSQL | `postgresql` | RETURNING clause, LIMIT/OFFSET |
| MySQL | `mysql` | LAST_INSERT_ID(), LIMIT/OFFSET |
| Oracle | `oracle` | Sequences, FETCH FIRST |
| SQLite | `sqlite` | last_insert_rowid(), LIMIT/OFFSET |

=== Dialect-Specific SQL

==== INSERT Operations

| Dialect | SQL |
|-----|-----|
| PostgreSQL | `INSERT INTO users (name) VALUES (?) RETURNING *` |
| MySQL | `INSERT INTO users (name) VALUES (?)` + `SELECT LAST_INSERT_ID()` |
| Oracle | `INSERT INTO users (id, name) VALUES (seq.nextval, ?) RETURNING id INTO ?` |
| SQLite | `INSERT INTO users (name) VALUES (?)` + `SELECT last_insert_rowid()` |

==== Pagination

| Dialect | SQL |
|-----|-----|
| PostgreSQL | `SELECT * FROM users LIMIT ? OFFSET ?` |
| MySQL | `SELECT * FROM users LIMIT ? OFFSET ?` |
| Oracle | `SELECT * FROM users FETCH FIRST ? ROWS ONLY` |
| SQLite | `SELECT * FROM users LIMIT ? OFFSET ?` |

=== Migrating from Database-Specific Code

==== Before: PostgreSQL-Specific

[source,java]
----
// PostgreSQL-specific SQL
String sql = "SELECT * FROM users LIMIT ? OFFSET ?";
```

==== After: Dialect-Agnostic

```groovy
// Framework auto-detects and generates appropriate SQL
objectClass("User") {
    table("users")
}
// Works for all supported dialects
----

=== Migration Tips for SQL Dialects

1. Test with target database dialect
2. Use `dialect "auto"` for automatic detection
3. Avoid database-specific SQL in custom handlers
4. Use framework abstractions when possible
5. Check SQL generation logs for dialect-specific output

=== Common Issues and Solutions

==== Issue 1: Table Not Found

**Error**: `Table 'users' not found`

**Causes**:
* Schema not configured
* Database name not specified in connection
* Case sensitivity mismatch

**Solution**:
[source,groovy]
----
connector {
    configuration {
        jdbcUrl = "jdbc:postgresql://localhost:5432/mydb"  // Include database name
    }
}
----

==== Issue 2: Type Mismatch

**Error**: `Cannot convert String to Integer`

**Causes**:
* Attribute type in schema doesn't match database column type
* Custom transformer logic error

**Solution**:
[source,groovy]
----
objectClass("User") {
    table("users")
    attribute("id") {
        sql { type INTEGER }  // Match database column type
    }
}
----

==== Issue 3: Primary Key Required

**Error**: `Primary key not specified for object class`

**Solution**:
[source,groovy]
----
objectClass("User") {
    table("users")
    attribute("id") {
        sql { type INTEGER; primaryKey true }
    }
}
----

==== Issue 4: SQL Syntax Error

**Error**: Database-specific SQL syntax error

**Causes**:
* Wrong dialect configured
* Custom SQL with wrong dialect

**Solution**:
[source,groovy]
----
connector {
    configuration {
        dialect = "postgresql"  // Match target database
    }
}
----

==== Issue 5: Connection Timeout

**Error**: `Connection timeout exceeded`

**Solutions**:
[source,groovy]
----
connector {
    configuration {
        connectTimeout = 30  // seconds
        socketTimeout = 60   // seconds
        maxPoolSize = 50     // Increase pool size
    }
}
----

== Troubleshooting

=== Common Migration Pitfalls

1. **Incomplete Schema Definition**
   - Missing primary keys
   - Incorrect column types
   - Missing attribute mappings

2. **SQL Dialect Mismatch**
   - Wrong dialect configured
   - Database-specific SQL in custom handlers

3. **Type Conversion Issues**
   - SQL NULL vs Java null
   - Precision loss in numeric types
   - String encoding issues

4. **Connection Problems**
   - Wrong JDBC URL
   - Missing database credentials
   - Pool size too small

### Resolution Strategy

1. **Check configuration**
   - Verify all required fields
   - Validate SQL dialect
   - Test connection independently

2. **Enable debugging**
   - Turn on query logging
   - Check framework logs
   - Review generated SQL

3. **Test incrementally**
   - Start with simplest configuration
   - Add complexity step by step
   - Test each operation individually

4. **Consult documentation**
   - Framework configuration guide
   - SQL dialect documentation
   - Type mapping reference
