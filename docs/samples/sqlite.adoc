= SQLite Sample Connector

== Overview

This sample demonstrates a SQLite connector implementation for file-based storage.

== JDBC Configuration

[source,groovy]
----
jdbcUrl = 'jdbc:sqlite:/path/to/database.db'
username = ''
password = ''
driverClassName = 'org.sqlite.JDBC'

dialect = 'auto'
----

== Dialect

SQLite uses `sqlite` dialect:

[source,groovy]
----
dialect = 'sqlite'
----

== Schema Definition

[source,groovy]
----
objectClass("User") {
    table("users")
    
    attribute("id") {
        sql {
            type INTEGER
            primaryKey true
            autoIncrement true
        }
    }
    
    attribute("username") {
        sql {
            type TEXT
            notNull true
            unique true
        }
    }
    
    attribute("email") {
        sql {
            type TEXT
            notNull true
        }
    }
    
    attribute("created_at") {
        sql {
            type TEXT
            defaultValue "CURRENT_TIMESTAMP"
        }
    }
    
    attribute("updated_at") {
        sql {
            type TEXT
        }
    }
    
    attribute("is_active") {
        sql {
            type INTEGER
            defaultValue 1
        }
    }
}
----

== Table Creation Script

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    email TEXT NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT,
    is_active INTEGER DEFAULT 1
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_created_at ON users(created_at);
```

== CRUD Operations

=== Create

[source,groovy]
----
def user = new User()
user.username = 'john.doe'
user.email = 'john@example.com'
connector.create(user)
----

SQL: `INSERT INTO users (username, email, created_at) VALUES (?, ?, ?)`

=== Read

[source,groovy]
----
def user = connector.get("User", "john.doe")
def allUsers = connector.getAll("User")
----

SQL: `SELECT * FROM users WHERE username = ?`

=== Update

[source,groovy]
----
def user = connector.get("User", "john.doe")
user.email = 'newemail@example.com'
connector.update(user)
----

SQL: `UPDATE users SET email = ? WHERE username = ?`

=== Delete

[source,groovy]
----
connector.delete("User", "john.doe")
----

SQL: `DELETE FROM users WHERE username = ?`

connectionPool {

    minimumIdle = 1

    idleTimeout = 600000

----

== Database-Specific Considerations

=== SQLite Features

1. **File-based**: Database is a single file
2. **No separate server**: No network overhead
3. **ACID compliant**: Full ACID compliance for single-file mode
4. **Limited concurrency**: Write operations block the entire database

=== Performance Tips

- Use WAL (Write-Ahead Logging) mode for better concurrency:
  ```groovy
  jdbcUrl = 'jdbc:sqlite:/path/to/database.db?journal_mode=WAL'
  ```
- Enable shared cache for multi-threaded access:
  ```groovy
  jdbcUrl = 'jdbc:sqlite:/path/to/database.db?cache=shared'
  ```
- Use transactions for batch operations
- Consider `PRAGMA synchronous=NORMAL` for better performance

=== Character Encoding

SQLite uses UTF-8 by default. No special configuration needed.

=== Common Issues

1. **Database locked errors**: Use WAL mode and proper transaction handling
2. **MAX_VARIABLE_NUMBER**: SQLite limits SQL variables to 999
   - Solution: Batch operations into smaller chunks
3. **No ALTER TABLE support for all operations**
   - Solution: Create new table, copy data, rename
