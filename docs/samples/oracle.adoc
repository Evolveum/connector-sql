= Oracle Sample Connector

== Overview

This sample demonstrates an Oracle Database connector implementation.

== JDBC Configuration

[source,groovy]
----
jdbcUrl = 'jdbc:oracle:thin:@localhost:1521:ORCLCDB'
username = 'oracle'
password = 'secret'
driverClassName = 'oracle.jdbc.OracleDriver'

dialect = 'auto'
----

== Table and Sequence Creation Script

```sql
CREATE TABLE users (
    id NUMBER PRIMARY KEY,
    username VARCHAR2(255) NOT NULL UNIQUE,
    email VARCHAR2(255) NOT NULL,
    created_at TIMESTAMP DEFAULT SYSTIMESTAMP,
    updated_at TIMESTAMP,
    is_active NUMBER(1) DEFAULT 1
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_created_at ON users(created_at);

CREATE SEQUENCE users_seq START WITH 1 INCREMENT BY 1;

CREATE OR REPLACE TRIGGER users_before_insert
BEFORE INSERT ON users
FOR EACH ROW
BEGIN
    IF :NEW.id IS NULL THEN
        SELECT users_seq.NEXTVAL INTO :NEW.id FROM DUAL;
    END IF;
END;
```

== CRUD Operations

=== Create

[source,groovy]
----
def user = new User()
user.username = 'john.doe'
user.email = 'john@example.com'
connector.create(user)
----

SQL: `INSERT INTO users (id, username, email, created_at) VALUES (?, ?, ?, ?)`

=== Read

[source,groovy]
----
def user = connector.get("User", "john.doe")
def allUsers = connector.getAll("User")
----

SQL: `SELECT * FROM users WHERE username = ?`

=== Update

[source,groovy]
----
def user = connector.get("User", "john.doe")
user.email = 'newemail@example.com'
connector.update(user)
----

SQL: `UPDATE users SET email = ? WHERE username = ?`

=== Delete

[source,groovy]
----
connector.delete("User", "john.doe")
----

SQL: `DELETE FROM users WHERE username = ?`

== Database-Specific Considerations

=== Oracle Features

1. **Sequences**: Oracle uses sequences for auto-increment values
2. **VARCHAR2**: Use `VARCHAR2` instead of `VARCHAR`
3. **NUMBER Type**: Oracle uses `NUMBER(p,s)` for numeric types
4. **DUAL Table**: Oracle requires `FROM DUAL` for select-only queries

=== Performance Tips

- Use connection pooling (Oracle connections are expensive)
- Consider statement caching: `useStatementPooling=true`
- Set `defaultFetchSize` for large result sets
- Use bind variables to prevent SQL injection and improve performance
- Monitor with `v$sql` and `v$session` views

=== Character Encoding

Oracle typically uses AL32UTF8 encoding. Ensure JDBC URL includes encoding:

```groovy
jdbcUrl = 'jdbc:oracle:thin:@localhost:1521:ORCLCDB?oracle.jdbc.defaultNChar=true'
```

=== Common Issues

1. **ORA-01795**: Maximum number of expressions in a list is 1000
   - Solution: Break large IN clauses into multiple OR clauses

2. **Date/Time handling**: Oracle uses `SYSDATE` and `SYSTIMESTAMP`
   - Use framework's timestamp handling for consistency

3. **Case sensitivity**: Oracle identifiers are case-insensitive unless quoted
   - Avoid case-sensitive identifiers unless necessary
