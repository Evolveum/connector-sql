[[dsl-overview]]
== Overview

The SQL Declarative Connector Framework provides a Groovy-based DSL for configuring and customizing database connectors. This reference documents all available DSL methods, configuration options, and building blocks.

The DSL is used in Groovy schema definitions and operation handlers to:

* Define database schema (tables, columns, relationships)
* Configure connector behavior (connection pooling, SQL dialect)
* Customize operation handlers (create, read, search, update, delete)
* Build dynamic filters and queries

[[dsl-architecture]]
== Architecture

The framework follows a layered architecture:

[cols="25,75"]
|===
| Layer | Description |
| Schema Layer | Groovy DSL for defining database schema and mappings |
| Handler Layer | Operation handlers generated from schema or custom implementations |
| SQL Layer | SQL statement building and dialect-specific generation |
| Connection Layer | Connection pooling and database interaction |
|===

[[dsl-context]]
== DSL Execution Context

The Groovy DSL runs in a context that provides access to:

* `context` - link:#sqlcontext[SqlContext] - Main configuration and schema context
* `schema` - link:#sqlschema[SqlSchema] - Database schema metadata
* `table(name)` - shortcut for schema.table(name)

=== Context Objects in Closures

[cols="1,2,2,3"]
|===
| Closure Type | Context Available | Description |
| Schema Definition | `context`, `schema` | Define object classes and attributes |
| Operation Handler | `context`, `schema`, `executeQuery`, `executeUpdate` | Custom operation logic |
| Filter Builder | `context`, `schema` | Build filter expressions |
|===

[[objectclass-builder]]

Define an object class mapping to a database table.

[source,groovy]

objectClass("User") {
    table("users")
    description "User account records"
    
    attribute("id") {
        sql { type INTEGER; primaryKey true }
    }
    
    attribute("username") {
        sql { type VARCHAR(255); nullable false }
        creatable true
        updatable true
    }
}
----

| `primaryKeys(names)` | List<String> | List of primary key attribute names |
| `creatable` | Boolean | Can create new records? |
| `updatable` | Boolean | Can update records? |
| `readable` | Boolean | Can read records? |

| `create { ... }` | Closure | Custom create handler |

[[operation-handlers]]
== Operation Handlers

[[search-handler]]
=== search { ... }

Customize search operation behavior.

[source,groovy]
----
search {
    handler { filter ->
        // Custom search logic
        if (filter?.startsWith("username=")) {
            def username = filter.substring(9)
            def sql = "SELECT * FROM users WHERE username = ?"
            return executeQuery(sql, [username])
        }
        return null  // Use default handler
    }
    
    returnAttributes ["id", "username", "email"]
}
----

| Option | Type | Description |
| `handler` | Closure | Custom search logic |
| `returnAttributes` | List<String> | Attributes to return |
| `defaultFilter` | Filter | Default filter when none provided |

[[handler-options]]
==== Handler Options

All operation handlers support these options:

| Option | Description |
| `handler` | Custom Groovy closure for operation logic |
| `returnAttributes` | Which attributes to return after operation |
| `generatedKeyAttribute` | Attribute name for auto-generated values |
| `whereClause` | Custom WHERE clause template |
| `options` | Operation-specific options map |

[[create-handler]]
=== create { ... }

Customize create operation behavior.

[source,groovy]
----
create {
    handler { attributes ->
        def sql = "INSERT INTO users (username, email) VALUES (?, ?) RETURNING id"
        def keys = executeInsert(sql, [attributes.username, attributes.email])
        return keys[0]
    }
    
    generatedKeyAttribute "id"
    returnAttributes ["id", "created_at"]
}
----

[[update-handler]]
=== update { ... }

Customize update operation behavior.

[source,groovy]
----
update {
    handler { attributes, filter ->
        def updates = []
        def params = []
        
        if (attributes.username) {
            updates << "username = ?"
            params << attributes.username
        }
        
        if (updates.isEmpty()) return
        
        def sql = "UPDATE users SET ${updates.join(', ')} WHERE id = ?"
        params << attributes.id
        
        executeUpdate(sql, params)
    }
}
----

[[delete-handler]]
=== delete { ... }

Customize delete operation behavior.

[source,groovy]
----
delete {
    handler { uid ->
        // Soft delete
        executeUpdate("UPDATE users SET is_active = 0 WHERE id = ?", [uid.value])
    }
}
----

=== Custom Search Handler

[source,groovy]
----
objectClass("User") {
    table("users")
    
    search {
        handler { filter ->
            def conditions = []
            def params = []
            
            if (filter?.isAnd()) {
                conditions << buildCondition(filter.left, params)
                conditions << buildCondition(filter.right, params)
            } else {
                conditions << buildCondition(filter, params)
            }
            
            def sql = "SELECT * FROM users WHERE ${conditions.join(' AND ')}"
            return executeQuery(sql, params)
        }
        
        def buildCondition(filter, params) {
            if (filter.isEqualTo()) {
                params << filter.value
                return "${filter.attributeName} = ?"
            } else if (filter.isLike()) {
                params << filter.value
                return "${filter.attributeName} LIKE ?"
            }
            return "1=1"
        }
    }
}
----

// Search with filter
def filter = new FilterBuilder()
    .and(
        new FilterBuilder().isEqualTo("is_active", true).build(),
        new FilterBuilder().isLike("username", "%john%").build()
    )
    .build()

connector.search(
    new ObjectClass("User"),
    filter,
    result -> {
        println result.getAttributeByName("username")?.value
    },
    null
)

// Update
connector.update(
    new ObjectClass("User"),
    new Uid("123"),
    Set.of(
        AttributeDeltaBuilder.build("email", "newemail@example.com")
    ),
    null
)

// Delete
connector.delete(
    new ObjectClass("User"),
    new Uid("123"),
    null
)

== Troubleshooting

=== Common Issues

| Issue | Solution |
| Schema not discovered | Enable `autoDiscoverSchema true` |
| Connection timeout | Increase `connectionTimeout` or `poolSize` |
| Wrong SQL dialect | Set explicit `dialect` value |
| Query errors | Enable `logSqlStatements true` |
| Type mapping issues | Verify link:#types-mapping[type mappings] |
